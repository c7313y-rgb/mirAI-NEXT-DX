# 副担任 mirAI NEXT｜v4.0

教員と生徒、それぞれに寄り添う教育実装OS。授業準備〜成果報告までを教員側で、カリキュラム閲覧・AI対話・キャリア探索を生徒側で、ひとつのプラットフォームに集約。

**ライブデモ:** https://c7313y-rgb.github.io/mirAI-NEXT-DEMO/

**提案:** 株式会社NOLTYプランナーズ ／ **サービス企画協力:** Edutex

---

## 🆕 v4.0 主な刷新点

- ✅ **教員 / 生徒の2ポータル**：ログイン画面分離、デモアカウントで即体験
- ✅ **教員ダッシュボード**：ホーム / 授業準備 / 授業実施 / 効果測定 / 進捗管理 / 生徒対話ログ / mirAI相談
- ✅ **生徒ポータル**：ホーム / カリキュラム閲覧 / mirAI対話 / キャリアシミュレーター
- ✅ **スライド自動生成 + PDFダウンロード**：jsPDF で授業当日に投影＆配布可能
- ✅ **実写画像12枚**：教員・生徒・教室・キャリアと、ワクワク感あふれるビジュアル
- ✅ **根拠不明データを全削除**：採択校数・効果率など、検証されていない数字は排除
- ✅ **副担任mirAI対話**：教員モード／生徒モードでスクリプトを分離、API接続も両対応

## 🎯 機能一覧

### 教員ポータル

#### 1. ホーム
- 準備済み授業数・実施完了数・生徒の相談数などのKPI
- 直近授業の一覧、クイックアクション

#### 2. 授業準備ウィザード（5ステップ）
| Step | 内容 |
|---|---|
| ① 入力 | テーマ・対象・目標・評価基準を入力（プリセット4種） |
| ② 生成 | 授業案・教材・ルーブリックをAIが下書き（タイプライター演出） |
| ③ レビュー | 3タブで編集→承認（再生成可能） |
| ④ スライド化 | 全自動でスライドデッキを構築（表紙含む10〜12枚） |
| ⑤ 出力・保存 | 実施予定日・対象クラスを設定してマイクラスに登録 |

- **スライドプレゼン再生機能**：フルスクリーン投影、←→キーで操作
- **PDFダウンロード**：jsPDF で授業当日に印刷可能なPDFを生成
- **授業案テキスト出力**：再編集用のテキストファイルも

#### 3. 授業実施
- 7項目のチェックリスト（導入→展開→まとめ→振り返り）
- 実施結果入力：参加率スライダー、ルーブリックA到達割合、教員所感

#### 4. 効果測定ダッシュボード
- KPI：出席率／参加体感率／A到達割合
- Chart.js による「学習ログ推移」「活動時間内訳」グラフ
- **教員所感を反映した自動改善提案**

#### 5. 進捗管理
- マイクラスに登録された全授業の一覧（状態・承認進捗）

#### 6. 生徒のmirAI対話ログ
- 進路相談・学習相談・悩みごとの3カテゴリでフィルタ
- 生徒の発言＋mirAIの応答をペアで表示

#### 7. mirAIに相談（教員モード）
- 授業設計・評価・成果報告など、教員向け応答スクリプト

### 生徒ポータル

#### 1. ホーム
- 受講中の単元・今週の課題・mirAI会話数・キャリア候補
- 進行中カリキュラムプレビュー、おすすめアクション

#### 2. カリキュラム
- 6つの単元カード（情報II / 探究 / データ活用 / 英語 / キャリア / PBL）
- 進捗バー、コマ数、教材数

#### 3. 副担任 mirAI と話す
- 学習・進路・悩みごとに寄り添う応答（生徒モード）
- 「両立がつらい」「進路に悩む」「探究テーマが決まらない」等

#### 4. キャリアシミュレーター ⭐ NEW
- 12の興味分野から複数選択（テクノロジー／医療／デザイン／環境 等）
- 12の進路候補からマッチング、上位4件を「マッチ度・必要スキル・学びの地図」付きで提示

## 🚀 GitHub Pages へのデプロイ

```bash
bash deploy.sh
```

- リポジトリをクローン
- 既存ファイルをバックアップ（`*_backup_YYYYMMDD_HHMMSS.html` 等）
- `index.html` / `style.css` / `app.js` / `README.md` / `assets/` をコピー
- `.nojekyll` を配置
- コミット & プッシュ

数十秒で https://c7313y-rgb.github.io/mirAI-NEXT-DEMO/ に反映されます。

## 🔐 認証について

このv4.0は **デモアカウント方式（事前定義）** です。
- パスワードはクライアント側で照合（実運用には不向き）
- 授業データは `localStorage` に保存（同一ブラウザでのみ保持）
- 本格運用には Firebase Auth + Firestore などへの移行が必要
